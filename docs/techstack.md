# Tech Stack

Технологічний стек навмисно мінімальний: **звичайний C++** плюс ті функції Win32 API, які вже задіяні в поточному коді `kursova/kursova.cpp` для консольних візуальних ефектів. Ніяких сторонніх бібліотек, фреймворків чи пакетних менеджерів.

## Мова

- **C++** — стандарт за замовчуванням MSVC (`v143`, Visual Studio 2022). Прапорці: `/EHsc`, Unicode character set.
- Цільова платформа: **Windows**, консольне застосування. Основна конфігурація — `x64`.

## Візуальні ефекти (консоль Windows)

Беруться **ті самі**, які вже використовуються в `kursova/kursova.cpp`:

- **`<windows.h>`** — Win32 API:
  - `GetStdHandle(STD_OUTPUT_HANDLE)` — дескриптор консолі.
  - `SetConsoleTextAttribute` — колір тексту та фону (використовується в `setColor` — `kursova/kursova.cpp:25`).
  - `SetConsoleCursorInfo` — приховання курсора (`hideCursor` — `kursova/kursova.cpp:16`).
  - `SetConsoleCP(1251)` / `SetConsoleOutputCP(1251)` — кодова сторінка CP1251 для коректного відображення кирилиці.
  - `SetConsoleCursorPosition` — **буде** додано для рендеру вузлів графа на 2D полі за координатами `(x, y)`.
- **`<conio.h>`** — `_getch` для зчитування клавіш без Enter (стрілки + Enter у меню; коди `VK_UP=72`, `VK_DOWN=80`, `VK_RETURN=13`, префікс `224` для розширених).
- **`system("cls")`** — очищення консолі між екранами.

Інших способів малювання чи анімації не задіюється — усе через ці примітиви.

## Стандартна бібліотека C++ (STL)

Те, що вже використано, і що плануємо далі:

- `<iostream>`, `<string>` — ввід/вивід, рядкові операції.
- `<vector>` — списки ребер, шляхи, пункти меню, буфери рендеру.
- `<unordered_map>` — вузли за `id` та список суміжності в `NetworkGraph`.
- `<queue>` — `priority_queue` для Дейкстри та A\*.
- `<limits>` — `numeric_limits<double>::infinity()` як початкова відстань.
- `<algorithm>` — `reverse` для відновлення шляху, плюс інші стандартні утиліти за потребою.
- `<chrono>` — очікуваний для заміру часу виконання алгоритмів (для звіту порівняння) та для керування затримкою анімації.

## Структура проєкту (файли)

Підхід — **змішаний**: невеликі утиліти залишаються header-only, великі фічі отримують власну пару `.h + .cpp`. Усе зводиться в `kursova/kursova.cpp` (точка входу).

### Поточні файли

- `kursova/kursova.cpp` — точка входу, консольний UI, головне меню.
- `kursova/NetworkGraph.h` — header-only клас `NetworkGraph` (вузли, ребра, список суміжності).
- `kursova/RoutingAlgorithms.h` — header-only клас `RoutingAlgorithms` зі статичними методами алгоритмів та структура `RoutingResult`.

### Орієнтовний перелік майбутніх модулів

Імена робочі, фінальні уточнюватимуться під час реалізації відповідних пунктів `roadmap.md`.

| Модуль | Тип | Відповідальність |
| --- | --- | --- |
| `ConsoleRenderer` | `.h + .cpp` | Рендер 2D поля консолі: позиціонування курсора, малювання вузлів/ребер, кольорові виділення (відвіданий / фронтир / шлях). Фіча 1.1, 3.1. |
| `GraphBuilder` | `.h + .cpp` | Інтерактивне додавання вузлів на 2D полі + текстове введення ребер. Фічі 1.2, 1.3. |
| `TopologyLibrary` | header-only | Колекція заготовлених топологій, які повертають готовий `NetworkGraph`. Фіча 1.4. |
| `AnimationController` | header-only | Керування затримкою між кроками та гарячими клавішами під час анімації (`+`/`-`, пауза). Фіча 3.3. |
| `PacketAnimator` | `.h + .cpp` | Анімація руху пакета вздовж знайденого шляху. Фіча 3.2. |
| `ComparisonReport` | `.h + .cpp` | Формування таблиці метрик + текстовий висновок після прогону трьох алгоритмів. Фіча 4.1. |
| `RoutingAlgorithms.h` (розширення) | header-only | Додавання `bellmanFord` та `aStar` до наявного класу. Фази 2.1, 2.2. |

## Збірка

Коротко — деталі вже є в `CLAUDE.md`:

- **Основний шлях** — MSBuild / Visual Studio 2022:

  ```
  msbuild kursova.sln /p:Configuration=Debug /p:Platform=x64
  ```

- **VS Code task** (`.vscode/tasks.json`) — `cl.exe /Zi /EHsc /nologo` по активному файлу. Працює, поки залежності header-only; при додаванні `.cpp` модулів — оновити задачу та `kursova/kursova.vcxproj`.

- **Запуск** — `./kursova/kursova.exe` (інтерактивне меню: стрілки + Enter).

## Чого НЕ використовуємо

Фіксую явно, щоб не виникало спокуси додати зайве:

- Жодних сторонніх бібліотек (Boost, fmt, ncurses, PDCurses, SFML тощо).
- Жодних пакетних менеджерів (vcpkg, Conan).
- Жодних кросплатформних шарів — проєкт свідомо Windows-only.
- Жодних тестових фреймворків — тестування ручне, через запуск застосунку.
