# ВИСНОВКИ

Під час розробки даного програмного продукту було реалізовано консольний симулятор комп'ютерної мережі, призначений для порівняльного аналізу трьох класичних алгоритмів маршрутизації: Дейкстри, Беллмана-Форда та A\*. Програма моделює мережу у вигляді неорієнтованого зваженого графа маршрутизаторів, надає користувачу можливість побудувати граф вручну або обрати одну із заготовлених топологій (Star, Mesh, Linear), послідовно запускає всі три алгоритми на одному й тому самому графі з покроковою візуалізацією, анімує передачу пакета знайденим маршрутом та формує порівняльну таблицю метрик із текстовим висновком про найефективніший алгоритм на конкретній топології.

Програма реалізована мовою програмування C++ у середовищі розробки Visual Studio 2022 за модульним підходом із розділенням на одну основну одиницю трансляції та шість header-only модулів: моделі графа, бібліотеки топологій, побудовника графа, базових утиліт консольного рендерингу, алгоритмів маршрутизації та модуля візуалізації. Інтерфейс реалізований в алфавітно-цифровому режимі консольного вікна за допомогою бібліотеки «windows.h» та елементів псевдографіки; рух по меню — за допомогою клавіш-стрілок, підтвердження вибору — клавішею Enter, керування анімацією — клавішами Space (пауза), Up/Down (швидкість) та Esc (пропуск). Для коректного відображення кирилиці встановлюється кодова сторінка CP1251 на вході та на виході консолі.

До переваг розробленої програми можна віднести: уніфіковану сигнатуру всіх трьох алгоритмів, що спрощує їх додавання та заміну; розділення етапу обчислення шляху від етапу його візуалізації через механізм покрокового журналу, що дозволяє відтворювати хід роботи алгоритму без повторного запуску; гнучке керування анімацією з п'ятьма рівнями швидкості, паузою та пропуском окремого алгоритму; стійкість до некоректного вводу з виведенням інформативних повідомлень без аварійного завершення програми; єдиний стиль оформлення всіх екранів з використанням спільної палітри кольорів, уніфікованих заголовків і підказок.

До обмежень розробленої програми належить наступне: підтримка лише неорієнтованих графів на рівні користувацького інтерфейсу, попри те, що внутрішня модель даних допускає й орієнтовані; обмеження розміру поля візуалізації величиною 80×20 символів, що звужує максимальний розмір наочно відображуваних топологій; відсутність механізму збереження та завантаження побудованого графа у файл, через що після закриття програми побудовану топологію потрібно відтворювати заново; фіксована кодова сторінка CP1251, яка робить програму залежною від консолі Windows.

У майбутньому планується розширення функціональності програмного продукту, зокрема: додавання алгоритму Флойда-Воршалла для обчислення найкоротших шляхів між усіма парами вершин графа; впровадження імпорту та експорту побудованого графа у текстовий файл для повторного використання збережених топологій; розширення бібліотеки топологій на орієнтовані графи з відповідним коригуванням користувацького інтерфейсу. Отже, поставлене завдання було повністю виконано, усі вимоги листа-завдання задоволені, а розроблений програмний продукт має потенціал для подальшого розвитку.
